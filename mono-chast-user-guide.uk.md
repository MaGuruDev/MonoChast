# MaGuru MonoChast — Посібник користувача

**Версія:** 1.1.0  
**Сумісність:** Adobe Commerce / Magento Open Source 2.4.4 – 2.4.x  
**PHP:** 8.1 або вище  
**Потребує:** MaGuru Monobank Core  
**Ліцензія:** Платно

---

## Огляд

MaGuru MonoChast додає **Monobank Частина** — спосіб оплати "купи зараз, плати пізніше" (BNPL) частинами — до вашого магазину Adobe Commerce. Клієнти з карткою Monobank можуть розділити вартість покупки на 3–24 щомісячні платежі без додаткових відсотків.

Замовлення з розстрочкою проходять через стандартний чекаут Magento з окремим варіантом оплати "Pay in installments". Все інше — схвалення, обробка платежу та керування розстрочкою — повністю на боці Monobank.

> **Фіскальні чеки:** якщо ви також використовуєте **MaGuru Monobank Fiscal**, майте на увазі, що зараз він фіскалізує лише замовлення **MaGuru Monobank Payment**. Замовлення розстрочки, оформлені через MonoChast, цим модулем автоматично не фіскалізуються.

---

## Що він робить

- Додає **Monobank Частина** як окремий спосіб оплати на чекауті
- Дозволяє налаштувати, які терміни розстрочки доступні (3, 6, 10, 12, 18 або 24 місяці)
- Підтримує опціональну валідацію телефону на чекауті — перевіряє право клієнта на розстрочку через Monobank до продовження оформлення
- Надає дії **Confirm Delivery** (підтвердити доставку) та **Reject Order** (скасувати замовлення) в перегляді замовлення в адмінці (вимагаються Monobank для активації або скасування розстрочки)
- Автоматично підтверджує доставку при створенні відвантаження в Magento (опціонально)
- Надає завантаження **Guarantee Letter (PDF)** — юридично зобов'язуючий документ від Monobank про угоду розстрочки
- Грід в адмінці з усіма замовленнями розстрочки та їхніми статусами Monobank

### Налаштування черги повідомлень (обов'язково)

Підтвердження замовлення при відвантаженні йде через чергу повідомлень Magento і **нічого не робить, доки не запущено consumer**; сам Magento не запускає consumer'и автоматично.

Додайте наступне в `app/etc/env.php`:

```php
'cron_consumers_runner' => [
    'cron_run' => true,
    'max_messages' => 200,
    'consumers' => [
        'mono_chast.order.confirm',
    ]
],
```

Далі cron самого Magento автоматично запускатиме цей consumer — жодного додаткового процесу не потрібно. Якщо цей крок пропустити, підтвердження замовлення Частинами при відвантаженні ніколи не дійде до Monobank, без жодної помилки де-небудь.

## Налаштування

Перейдіть у **Stores → Configuration → Sales → Payment Methods → Monobank Chast**.

> Середовище API Monobank Chast (Production / Sandbox / Stage) встановлюється в **Stores → Configuration → MaGuru → Monobank Integration → MonoChast**.

---

### Загальні налаштування

| Параметр | Опис | За замовчуванням |
|---|---|---|
| **Enable** | Активує Monobank Частина на чекауті | No |
| **Title** | Назва, яку бачить клієнт на чекауті (підпис радіо-кнопки) | Monobank Частинами (BNPL) |
| **Checkout Button Label** | Текст на кнопці підтвердження оплати на чекауті | Pay in installments |
| **Available Parts** | Мультивибір дозволених термінів розстрочки (місяців): 3, 6, 10, 12, 18, 24 | Усі |
| **Min Order Total (UAH)** | Мінімальна сума замовлення для показу цього способу оплати (залиште порожнім — без мінімуму) | — |
| **Max Order Total (UAH)** | Максимальна сума замовлення для показу цього способу оплати (залиште порожнім — без максимуму) | — |
| **Auto-confirm on Shipment** | Автоматично підтверджує доставку в Monobank при створенні відвантаження Magento | Yes |
| **Validate Client** | Пропонує клієнту ввести номер телефону і перевіряє право на розстрочку через Monobank до продовження чекауту | No |
| **Debug Logging** | Записувати детальні дані API в лог | No |

## Досвід клієнта на чекауті

1. На чекауті клієнт обирає **Pay in installments** як спосіб оплати
2. Якщо увімкнено **Validate Client**, він вводить свій номер телефону Monobank; модуль перевіряє право на розстрочку в реальному часі
3. Клієнт обирає бажану кількість платежів (з налаштованих вами варіантів)
4. Його перенаправляють у Monobank для підтвердження угоди розстрочки
5. Після повернення замовлення створюється в Magento, а оновлення статусу надходять через webhook і фонове опитування

## Перегляд замовлення в адмінці

Коли замовлення оформлене через MonoChast, на сторінці перегляду замовлення з'являється блок **Monobank Chast**, що показує:

- Mono Order ID, статус, підстатус, кількість платежів, суму (UAH) та телефон клієнта

**Кнопки дій** (показуються залежно від поточного статусу):

| Кнопка | Коли видима | Що робить |
|---|---|---|
| **Confirm Delivery** | Sub-state = `WAITING_FOR_STORE_CONFIRM` | Підтверджує в Monobank, що товар доставлено; активує угоду розстрочки |
| **Reject Order** | State = `IN_PROCESS` | Скасовує замовлення розстрочки в Monobank |
| **Download Guarantee Letter (PDF)** | Є Mono Order ID (будь-який статус) | Завантажує юридичний PDF-документ, що підтверджує угоду розстрочки |
| **View Guarantee Letter Data (JSON)** | Є Mono Order ID (будь-який статус) | Відкриває структуровані дані гарантійного листа — ім'я клієнта, паспортні дані, умови банківської угоди — в новій вкладці |
| **Sync State** | Є Mono Order ID | Примусово оновлює статус з API Monobank негайно |

## Статуси замовлення розстрочки

Перегляд замовлення в адмінці, грід Chast Orders та коментарі до замовлення показують зрозумілі підписи (наприклад, "Exceeded installment limit") замість сирих кодів Monobank — таблиця нижче наводить базові коди для довідки.

| State | Значення |
|---|---|
| **IN_PROCESS** | Замовлення створене в Monobank, очікує підтвердження клієнта на його пристрої |
| **SUCCESS** | Замовлення завершене — розстрочка активна й діє |
| **FAIL** | Замовлення було відхилене, скасоване або не вдалося |
| **EARLY_CLOSED_BY_CLIENT** | Клієнт достроково закрив угоду розстрочки |

**Підстатуси IN_PROCESS:**

| Sub-State | Значення |
|---|---|
| **WAITING_FOR_CLIENT** | Очікується підтвердження клієнта в застосунку Monobank |
| **WAITING_FOR_STORE_CONFIRM** | Розстрочку схвалено Monobank; магазин має підтвердити доставку, щоб продовжити |

**Підстатуси FAIL** (чому замовлення не вдалося і що робити):

| Sub-State | Значення | Рекомендована дія |
|---|---|---|
| **CLIENT_NOT_FOUND** | Клієнта не знайдено або він не є клієнтом Monobank | Перевірити формат номера телефону |
| **EXCEEDED_SUM_LIMIT** | Клієнт перевищив свій ліміт на розстрочку | Попросити клієнта зменшити суму замовлення або використати інший спосіб оплати |
| **EXISTS_OTHER_OPEN_ORDER** | У клієнта вже є інше незавершене замовлення розстрочки | Почекати 15 хвилин або попросити клієнта скасувати існуюче замовлення |
| **NOT_ENOUGH_MONEY_FOR_INIT_DEBIT** | Недостатньо коштів для першого платежу | Клієнту потрібно поповнити картку |
| **REJECTED_BY_CLIENT** | Клієнт відхилив пропозицію розстрочки | Клієнт може спробувати пізніше або використати інший спосіб оплати |
| **PAY_PARTS_ARE_NOT_ACCEPTABLE** | Запитана кількість платежів неприйнятна для Monobank | Звернутись у підтримку Monobank |
| **FRAUD_REJECTED** | Відхилено системою запобігання шахрайству Monobank | Звернутись у підтримку Monobank |
| **RESTRICTED_BY_RISKS** | Обмежено системою управління ризиками Monobank | Звернутись у підтримку Monobank |
| **CLIENT_PUSH_TIMEOUT** | Клієнт не відповів протягом 15 хвилин | Зв'язатись з клієнтом або попросити повторити спробу |
| **REJECTED_BY_STORE** | Магазин скасував замовлення | Немає — це було навмисно |
| **FAIL** | Внутрішня помилка Monobank | Повторити через кілька хвилин |

## Підтвердження доставки

Monobank вимагає від магазинів явно підтвердити, що товар або послугу доставлено, перш ніж угода розстрочки активується. Є два способи це зробити:

**Варіант A — Автоматично (рекомендовано):** увімкніть **Auto-confirm on Shipment** у конфігурації. Коли ви створюєте відвантаження Magento для замовлення, модуль автоматично надсилає підтвердження доставки в Monobank.

**Варіант B — Вручну:** коли статус замовлення досягає `WAITING_FOR_STORE_CONFIRM`, натисніть кнопку **Confirm Delivery** в перегляді замовлення в адмінці.

> Поки доставку не підтверджено, угода розстрочки не активована, і клієнт формально не взяв на себе графік платежів.

## Часті запитання

**Q: Що таке "Auto-confirm on Shipment" і чи варто його вмикати?**

Коли увімкнено, створення відвантаження Magento для замовлення розстрочки автоматично надсилає підтвердження доставки в Monobank. Це рекомендоване налаштування для фізичних товарів. Якщо ви продаєте послуги чи цифрові товари, що доставляються миттєво, можливо, зручніше підтверджувати доставку вручну кнопкою в адмінці.

---

**Q: Клієнт каже, що його розстрочку схвалено, але замовлення досі "в очікуванні" в моєму магазині. Що робити?**

Натисніть кнопку **Sync State** у перегляді замовлення в адмінці, щоб отримати актуальний статус від Monobank. Якщо статус `WAITING_FOR_STORE_CONFIRM`, підтвердіть доставку кнопкою або створенням відвантаження (якщо увімкнено Auto-confirm). Якщо проблема лишається, зачекайте фоновий cron (кожні 2 хвилини) або запустіть `bin/magento mono:chast:status-check` з командного рядка.

---

**Q: Що таке Guarantee Letter?**

Guarantee Letter — це юридично зобов'язуючий PDF-документ, виданий Monobank, що підтверджує угоду розстрочки між банком і клієнтом. Він доступний для завантаження з перегляду замовлення в адмінці, щойно Monobank призначить Mono Order ID (є також кнопка **View Guarantee Letter Data (JSON)** для структурованих даних). Можливо, вам доведеться надати цей документ клієнту на його запит.

---

**Q: Чи можу я обмежити, які терміни розстрочки пропонуються?**

Так. У налаштуванні мультивибору **Available Parts** оберіть тільки ті терміни, які хочете пропонувати. Наприклад, якщо хочете пропонувати тільки плани на 6 і 12 місяців — оберіть лише ці два варіанти.

---

**Q: Чи можу я встановити мінімальну суму замовлення для оплати розстрочкою?**

Так. Введіть значення (у гривнях) у **Min Order Total**. Замовлення нижче цієї суми не показуватимуть опцію оплати Monobank Частина на чекауті.

---

**Q: Яке середовище використовувати під час тестування?**

Встановіть середовище MonoChast на **Sandbox** у **Stores → Configuration → MaGuru → Monobank Integration → MonoChast** і використайте Sandbox Store ID і Secret від Monobank. Це дозволяє повне наскрізне тестування без реальних фінансових транзакцій.

---

**Q: Клієнт передумав після схвалення розстрочки. Чи можу я її скасувати?**

Так. Поки замовлення в статусі `IN_PROCESS` (до підтвердження доставки), ви можете натиснути **Reject Order** у перегляді замовлення в адмінці, щоб скасувати розстрочку в Monobank.

---

## Підтримка

Перед зверненням у підтримку зберіть:

1. Номер замовлення Magento (Order #) і Mono Order ID з гріда Chast Orders

3. Вміст `var/log/mono_chast.log`, якщо увімкнено Debug Logging
4. Чи стосується проблема всіх замовлень розстрочки, чи конкретних
